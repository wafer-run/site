Creating a Block

There are two ways to create blocks: WASM blocks written in any supported language and loaded at runtime, or native Rust blocks that compile directly into your binary. Both share the same wafer-block types (Message, BlockResult, BlockInfo, etc.).

WASM Blocks Native Rust Blocks
Languages Rust, Go, TypeScript Rust only
Sandboxed Yes — capability model, requires list No — full OS access
Deployable to CF Yes No — local only
Build step wafer build cargo build (included in binary)
Use when Business logic, third-party integrations, tenant code Infrastructure, platform services, full performance

WASM Blocks

WASM blocks are the recommended path for most use cases. They are sandboxed, deployable to Cloudflare Workers, and support multiple languages. Use the wafer CLI to scaffold and build them.

1. Scaffold a Block Project

wafer new myorg/greet --lang rust
cd myorg-greet
wafer new myorg/greet --lang go
cd myorg-greet
wafer new myorg/greet --lang ts
cd myorg-greet

This creates a project with a manifest.json, source file, and test fixtures.

2. Edit Your Block

// src/lib.rs
use wafer_sdk::*;

struct GreetBlock;

#[wafer_block(
    name = "myorg/greet",
    version = "0.1.0",
    interface = "handler@v1",
    summary = "Returns a greeting"
)]
impl GreetBlock {
    fn handle(msg: Message) -> BlockResult {
        json_respond(&msg, &serde_json::json!({"greeting": "hi there"}))
    }
}
// main.go
package main

import wafer "github.com/wafer-run/wafer-sdk-go"

type GreetBlock struct{}

func (b *GreetBlock) Info() wafer.BlockInfo {
    return wafer.BlockInfo{
        Name:      "myorg/greet",
        Version:   "0.1.0",
        Interface: "handler@v1",
        Summary:   "Returns a greeting",
    }
}

func (b *GreetBlock) Handle(ctx *wafer.Context, msg *wafer.Message) *wafer.Result {
    return wafer.JSONRespond(msg, map[string]string{"greeting": "hi there"})
}

func (b *GreetBlock) Lifecycle(ctx *wafer.Context, event wafer.LifecycleEvent) error {
    return nil
}

func main() {
    wafer.Export(&GreetBlock{})
}
// src/index.ts
import { block, Message, BlockResult } from "@wafer-run/sdk";

export default block({
    name: "myorg/greet",
    version: "0.1.0",
    interface: "handler@v1",
    summary: "Returns a greeting",
    handle(msg: Message): BlockResult {
        return msg.jsonRespond({ greeting: "hi there" });
    },
});

3. Build and Test

wafer build   # compile to target/block.wasm
wafer test    # run tests/ fixtures against the block

4. Place in blocks/ for Auto-Discovery

Solobase scans blocks/**/target/block.wasm on startup. Copy or build your block inside the blocks/ directory and restart Solobase:

cp -r myorg-greet /path/to/solobase/blocks/
# or build in place
cd /path/to/solobase/blocks/myorg-greet && wafer build

See WASM Blocks for the full manifest.json format and auto-discovery details.

Native Rust Blocks

A native block is a Rust struct that implements the Block trait and is compiled directly into your binary. No CLI needed — just implement the trait and register it at startup. Native blocks have full OS access but cannot be deployed to Cloudflare Workers.

1. Implement the Block Trait

// src/blocks/greet.rs
use wafer_run::*;

pub struct GreetBlock;

#[async_trait::async_trait]
impl Block for GreetBlock {
    fn info(&self) -> BlockInfo {
        BlockInfo {
            name: "myorg/greet".to_string(),
            version: "1.0.0".to_string(),
            interface: "handler@v1".to_string(),
            summary: "Returns a greeting".to_string(),
            instance_mode: InstanceMode::Singleton,
            allowed_modes: Vec::new(),
            admin_ui: None,
            runtime: BlockRuntime::Native,
            requires: Vec::new(),
        }
    }

    async fn handle(&self, _ctx: &dyn Context, msg: &mut Message) -> Result_ {
        json_respond(msg, &serde_json::json!({"greeting": "hi there"}))
    }

    async fn lifecycle(&self, _ctx: &dyn Context, _event: LifecycleEvent)
        -> std::result::Result<(), WaferError> { Ok(()) }
}

2. Register It

// main.rs
use std::sync::Arc;

let mut w = Wafer::new();
w.register_block("myorg/greet", Arc::new(GreetBlock))?;

register_block() returns Result<(), String> and rejects duplicate block names. Block names use the same {org}/{block} format as WASM blocks.

For simple blocks you can skip the struct and register an inline handler:

w.register_func("myorg/greet", |_ctx, msg| {
    json_respond(msg, &serde_json::json!({"greeting": "hi there"}))
})?;

Native blocks compile directly into your binary — no .wasm file, no runtime loading, zero overhead.
