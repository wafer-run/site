Quick Start

Create and run your first WASM block in under five minutes using the wafer CLI.

1. Install the CLI

The wafer CLI is a native Rust binary. Install it with cargo:

cargo install wafer-cli

Verify the installation:

wafer --version

2. Scaffold a Block Project

Use wafer new to scaffold a block project. Specify your org/block name and language:

wafer new myorg/hello --lang rust
cd myorg-hello
wafer new myorg/hello --lang go
cd myorg-hello
wafer new myorg/hello --lang ts
cd myorg-hello

This creates a project with a manifest.json, source file, and test fixtures.

3. Edit the Handler

Open the generated source file and edit the handle function:

// src/lib.rs
use wafer_sdk::*;

struct HelloBlock;

#[wafer_block(
    name = "myorg/hello",
    version = "0.1.0",
    interface = "handler@v1",
    summary = "Says hello"
)]
impl HelloBlock {
    fn handle(msg: Message) -> BlockResult {
        json_respond(&msg, &serde_json::json!({"hello": "world"}))
    }
}
// main.go
package main

import wafer "github.com/wafer-run/wafer-sdk-go"

type HelloBlock struct{}

func (b *HelloBlock) Info() wafer.BlockInfo {
    return wafer.BlockInfo{
        Name:      "myorg/hello",
        Version:   "0.1.0",
        Interface: "handler@v1",
        Summary:   "Says hello",
    }
}

func (b *HelloBlock) Handle(ctx *wafer.Context, msg *wafer.Message) *wafer.Result {
    return wafer.JSONRespond(msg, map[string]string{"hello": "world"})
}

func (b *HelloBlock) Lifecycle(ctx *wafer.Context, event wafer.LifecycleEvent) error {
    return nil
}

func main() {
    wafer.Export(&HelloBlock{})
}
// src/index.ts
import { block, Message, BlockResult } from "@wafer-run/sdk";

export default block({
    name: "myorg/hello",
    version: "0.1.0",
    interface: "handler@v1",
    summary: "Says hello",
    handle(msg: Message): BlockResult {
        return msg.jsonRespond({ hello: "world" });
    },
});

4. Build to WASM

wafer build

The CLI compiles your block to WebAssembly with validation. Output lands in target/block.wasm.

5. Run Tests

wafer test

Test fixtures from the tests/ directory are run against the compiled block. Each fixture is a JSON file with input and expected fields.

6. Place in blocks/ Directory

Solobase auto-discovers WASM blocks by scanning blocks/**/target/block.wasm on startup. Copy or symlink your project into the blocks/ directory:

# from your Solobase project root
cp -r ../myorg-hello blocks/myorg-hello
# or build in place
cd blocks/myorg-hello && wafer build

7. Run Solobase

Start Solobase — it will auto-discover and load your block:

cargo run   # or ./solobase
[INFO] Discovered block: myorg/hello (blocks/myorg-hello/target/block.wasm)
[INFO] Registered 1 WASM block(s)
